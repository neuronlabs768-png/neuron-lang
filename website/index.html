Designed for ML Safety & Correctness

No more chasing silent numerical bugs, data leakage, or invalid causal claims through thousands of log lines.

Temporal Types

Prevents lookahead bias at compile time. Data variables carry temporal direction properties (e.g., past_to_future), and any index access into future timestamps is rejected by the type checker.

Causal Inference Primitives

Treats do-calculus as a first-class feature. Distinguishes observational conditioning observe() from intervention intervene() to prevent correlation/causation fallacies at compile time.

First-Class Uncertainty

Supports Uncertain[Tensor] types. Uncertainty bounds are propagated through mathematical operations automatically. The compiler issues a warning if confidence parameters are not explicitly checked.

Natively Differentiable

No libraries required. Functions are automatically differentiable, utilizing a tape-based reverse-mode autograd engine built directly into the compiler AST lowerer and VM backend.

Under the Hood: How NEURON Works

A native language compiler workflow, ensuring safety and extreme JIT execution performance.

01

Source Code

User writes .nr files using differentiable models, matrix multiplication, and causal primitives.

02

Type Checking

Checks tensor sizes, checks lookahead bounds, validates causal observations, and verifies effect flags.

03

AST to IR Lowering

Lowers code into a compiler Control Flow Graph (CFG) with custom basic blocks and tape nodes.

04

JIT Compiler

Generates host-native C code dynamically compiled into a DLL, linking directly for native-level execution speeds.

Key Math Engine Optimizations

IKJ Loop Reordering

Rearranges memory access patterns in matrix multiplication for sequential cache line retrieval, enabling CPU auto-vectorization into AVX2 instructions.

Thread-Local Allocator Pool

Thread-local caching allocator (Buffer) recycles intermediate vectors, completely eliminating dynamic allocation locks during backpropagation.

Bounds Check Elimination

Enables LLVM compiler optimization to prove indexing limits, removing runtime safety check overhead in matmul loops.

Powi Loop LICM

Moves exponentiation calculations outside the element-wise optimization steps, reducing loop evaluation times.

Frequently Asked Questions

Find answers to common technical and licensing questions regarding NEURON.

A library wrapper cannot verify mathematical properties at the syntax level. In Python, shape errors, temporal lookahead bias, and causal mode confusion only manifest as runtime exceptions or silent numerical calculation discrepancies. By writing a custom compiler in Rust, NEURON validates the structure of your network graph before compiling or executing it.

NEURON implements temporal direction analysis during type verification. When you construct a Temporal[T, past_to_future] reference, the compiler tracks historical sequences. Any index offset referencing a timestamp index higher than the current index (e.g. .after(k)) is caught by the compiler type-checker and rejected with a TemporalLeak compilation error.

The compiler (neuronc) is written in Rust and runs on Windows, macOS, and Linux. The JIT compiler requires clang or another host compiler to compile the dynamically linked native modules. Native code generation targets standard x86_64 AVX2 instruction sets.

The BSL allows 100% free use of the compiler and runtime for development, testing, research, and non-production deployments. For proprietary commercial workloads (such as algorithmic trading models), it requires a commercial license. This ensures sustainable funding for the development team while guaranteeing that the code becomes completely open-source (Apache 2.0) on January 1, 2030.
